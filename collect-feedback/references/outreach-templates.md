# Outreach Templates

Structural patterns for feedback outreach messages. Customize these for each project — never send generic messages.

## Core Principles

Every outreach message must:
1. Reference the specific problem (not the product)
2. Set time expectations ("5 minutes")
3. Set quality expectations ("rough," "just built," "MVP")
4. Ask for criticism, not praise ("what's broken" not "what do you think")
5. Stay under 80 words (people don't read long DMs from strangers)

## Reddit DM Pattern

```
Hey — saw your post about [THEIR SPECIFIC PROBLEM from their actual post].
I just built something that [ONE SENTENCE about what it does].
It's rough but the core [FEATURE] works.

Would you try it for 5 minutes and tell me what's confusing or missing?
Not selling anything — just want honest feedback from someone who actually
has this problem.

[LINK]
```

Why this works:
- References their post → they know you read it, not mass-DMing
- "Just built" → sets expectations low
- "Confusing or missing" → asks for specific criticism
- "Not selling" → removes sales resistance
- Under 60 words of actual content

## Community Post Pattern

```
Just built an MVP — [ONE SENTENCE what it does for whom].
Looking for 3 people who [HAVE THE SPECIFIC PROBLEM] to try it
for 5 minutes.

Not looking for "looks great!" — I need "this confused me" and
"I wanted to do X but couldn't."

DM me if you're in.

[LINK]
```

Why this works:
- "3 people" → scarcity, makes it feel selective
- Explicitly rejects positive feedback → signals maturity
- "DM me" → lower commitment than clicking a link publicly
- Works in Discord, Slack, forums, Indie Hackers

## Direct Message Pattern (someone you know)

```
Hey — I built something this morning that might help with
[THING THEY COMPLAINED ABOUT]. Can I send you a link?
I need 5 minutes of brutal honesty.
```

Why this works:
- Personal, not templated
- References a real conversation you had
- "Brutal honesty" → permission to be critical
- Under 40 words

## Anti-Patterns (do NOT do these)

**Too long:**
"Hey! I've been working on this project for a while and I think you might find it interesting. It's a tool that helps freelancers manage their invoicing workflow by automating..." → Nobody reads past the first sentence.

**Too salesy:**
"I noticed you're struggling with invoicing — I built the perfect solution! Check it out at..." → Sounds like spam. Instant ignore.

**Too vague:**
"Would you mind checking out my project and giving feedback?" → No context on what it is, what you want, or how long it takes. Easy to say no.

**Asking for validation, not criticism:**
"I built this thing, do you think it's good?" → They'll say yes to be nice. Useless.
